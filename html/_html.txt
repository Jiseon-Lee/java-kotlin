HTML(HyperText Markup Language)
 - 웹브라우저에서 해석하여 보여주는 언어로 웹의 기본이 됨
 	1990년대에 4.0이 2010년대에 5.0이 발표됨
 - 문법이 타 언어들에 비해 느슨한 편으로 오류가 발생하지 않고, 오동작을 하게 됨
 - 태그(tag)라는 명령어들의 집합으로 <태그명> 내용 부분 </태그명>의 태그 쌍으로 이루어져 있음

XML(eXtensible Markup Language)
 - 웹상에서 구조화된 문서나 데이터를 전송할 수 있게 설계된 표준화된 텍스트
 - 주로 데이터의 저장이나 전송등에 사용됨
 - 데이터를 전송해야 하므로 데이터들의 입력형식을 상당히 까다롭게 검사하는 방식(문법에 위배되면 오류 발생)

XHTML(eXtensible HyperText Markup Language)
 - HTML에 XML이 가진 특성중 문법에 대한 특성을 가미한 것
 - XHTML1.0은 HTML4.0과 거의 같으나 문법 오류를 표시하는 것에서 차이가 있음

웹 표준화(Web Standard)
 - 2000년대 초반 발표된 XHTML을 기반으로 웹에서 사용되는 언어들을 표준화하여 지키자는 운동
 - 모든 사이트가 모든 브라우저에서 동일하게 보이고, 동일하게 동작하게 하자는 의미
 	단, 브라우저별로 다르게 보이는 부분(컨트롤 등)은 어쩔 수 없음
 - 모든 사용자(시각장애인 포함)들이 웹사이트를 편하게 사용하도록 하자는 의미도 포함
	시각장애인 전용 리더기로 사이트의 내용을 읽어줄 수 있음
	단, html 태그를 정확하게 웹표준에 맞게 사용해야 리더기가 정확하게 읽어줄 수 있음
 - 웹표준의 기본은 의미에 맞는 태그를 사용하자임
※ table 태그를 쓰지 않고 div 태그를 쓰는 것이 웹표준은 아님

태그(tag)
 - html 명령어로 특별한 경우가 아니면 무조건 영문 소문자로 입력함
 - html 문서를 이루는 요소라는 의미로 element라고 표현하기도 함(자바스크립트에서 주로 사용되는 용어이자 명령어)
 - "<"로 시작하여 ">"로 끝나는 html의 명령어로 보통 여는 태그와 닫는 태그의 쌍(pair)으로 구성되어 있음
 	예) <태그명> ... 내용 ... </태그명>
	단, 몇몇 태그는 닫는 태그가 따로 없어서 하나의 태그에서 열고 닫는 기능까지 모두 구현해야 함
	예) <태그명 />
 - 대부분의 태그는 속성이라는 것을 가지고 있어 태그에 여러 가지 부가적인 기능을 부여할 수 있음
 	문법 : <태그명 속성명1="값1" 속성명2="값2" ... 속성명n="값n"> - 속성값을 큰 따옴표로 묶어서 표현
 - 태그가 차지하는 위치와 영역에 따른 분류
 	인라인 요소(inline element) : 태그를 사용한 후 뒤의 내용이 줄이 바꾸지 않는 태그들을 의미
	 - 인라인 요소 안에서는 블록 요소를 사용할 수 없음
	 - 인라인 요소안에서는 인라인 요소를 사용할 수 있음
	블록 요소(block element) : 태그를 사용한 후 뒤의 내용이 줄이 바뀌어 다음 줄로 내려가는 태그들을 의미
	 - 돌깁적인 형태의 상자(box)를 의미하며 블록요소와 블록요소 사이에는 약간의 여백이 존재하며 그 여백은 브라우저마다 조금씩 다를 수 있음(CSS를 통해 여백을 동일하게 조절해야 함)
	 - 블록 요소 안에서는 인라인 요소 뿐만 아니라 블록 요소도 사용할 수 있음
 - html 문서의 확장자는 'html' 또는 'htm' 중 하나를 사용하면 되며 보통 'html'을 사용
 - html 문서내에서는 띄어쓰기가 한 번만 적용되며(여러 번 사용해도 그 중 하나만 적용됨), 여러 번을 띄우고 싶으면 &nbsp;을 사용해야 함
 - html 문서내에서 엔터는 한 번의 띄어쓰기로 해석되며, 다음 줄로 내리려면 <br/>이라는 태그를 사용해야 함

공용태그
1. <html>태그
 - html문서의 시작과 끝을 의미하는 태그
 	<html>부터 </html>까지가 하나의 웹문서이나 해당 범위 밖에서 임력해도 동작은 됨
 - 모든 태그들은 기본적으로 <html>태그 영역 내부에서 위치해야 함
 - <html>태그 내부에는 <head>와 <body> 두 개의 태그가 존재함

2. <head> 태그
 - 해당 문서의 윗부분에 존재하며 화면에 보이는 것보다 보이지 않는 작업이 더 많은 태그
 - 검색을 용이하게 하거나 문서에 대한 설명이나 정보를 입력하는 <meta> 태그 입력
 - 제목표시줄에 특정 문자열을 보여주는 <title> 태그 입력
 - 스타일 시트를 정의하거나 불러오는 <style> 태그 입력
 - 자바스크립트를 정의하거나 불러오는 <script> 태그 입력
 	<style>이나 <script> 모두 <head>가 아닌 <body> 태그 안에서도 원하는 위치에서 사용할 수 있으나 보통은 보기 편한 <head> 태그 안이나 양이 많을 경우 <body>의 아랫쪽에 입력하기도 함

3. <body> 태그
 - 웹브라우저에서 실제로 사용자에게 보여질 부분에 대한 영역으로 대부분의 작업이 이루어지는 곳
 - 기본적으로 여백을 가지고 있어 사이트의 컨텐츠가 브라우저에 딱 붙지 않게 해줌
	단, 보통 스타일 시트로 여백을 제거한 후 사용하게 됨



html에서의 색상값(css와 js에서도 동일하게 사용됨)
 - 영문으로 된 색상명 : white, black, red, yellow, blue, pink, lightgreen 등
 - 또는 총 6자리의 16진수로 된 RGB값으로 각 색상별로 두 자리씩 표현
 	16진수 : 한 자리의 숫자로 16가지 값을 표현하는 진법으로 0~9와 A~F로 표현
	16진수 두 자리로 색상을 각각 표현하여 전체 색상을 지정하게 됨
	예) #000000 : 검정, #ff0000 : 빨강, #00ff00 : 초록, #0000ff : 파랑, #ffffff : 흰색
	#은 붙이지 않아도 동작되나 css등에서는 제대로 동작하지 않을 수도 있으므로 반드시 붙여서 사용하는 것이 좋다.
	각 색상별 자릿수인 2자리는 00부터 ff까지 표현되며 10진수로 바꾸면 0~255까지 표현됨
	색상별 값이 동일하면 하나 씩만 표현해도 됨(ff0000 -> f00)
